2.2.1.6

- Reorganizada interface do SettingsPlugin: adicionados dois widgets colapsaveis - "Geral" (botao de preferencias, idioma, precisao de campos vetoriais, limiar assincrono) e "Plugin de Calculos Vetoriais" (metodo de calculo vetorial) para melhor organizacao da UI.

2.2.5.1

- Corrigido bug de inicializacao de log em `LogUtils`: anteriormente usava `Path.cwd()` (ex: `C:\Windows\system32`) quando nao inicializado, causando `PermissionError` ao criar pasta `log`.
- Implementado fallback seguro para `plugin_root` via `Path(__file__).resolve().parents[3]` e, se necessario, `Path.home()/Cadmus/log`.
- Ajustado `LogUtils._write_event` para continuar sem arquivo de log e enviar mensagem por `QgsMessageLog`/stderr em casos de falha de criacao de arquivo.

2.2.5.2

- Adicionados no `SettingsPlugin`, dentro de `calc_collapsable`, dois campos configuraveis inicialmente para nomes de area cartesiana e elipsoidal com `WidgetFactory.create_input_fields_widget`.
- As preferencias desses campos passaram a ser carregadas e salvas em `ToolKey.VECTOR_FIELDS`.
- Incluida validacao para impedir salvamento quando os dois valores configurados forem iguais, exibindo alerta com `QgisMessageUtil`.

2.2.5.3

- Ajustados os dois campos do `SettingsPlugin` para representar sufixos, nao nomes completos de campo: `Sulfixo Cartesiano` com padrao `""` e `Sulfixo Elipsoidal` com padrao `"_eli"`.
- As preferencias em `ToolKey.VECTOR_FIELDS` passaram a usar as chaves `cartesian_suffix` e `ellipsoidal_suffix`.
- Mantida a validacao para impedir salvamento quando os dois sufixos forem iguais, agora com mensagem alinhada ao novo comportamento.

2.2.5.4

- Implementada a leitura dos sufixos configurados em `ToolKey.VECTOR_FIELDS` no `VectorFieldsCalculationPlugin`.
- Os calculos de `length` para linhas e `area` para poligonos agora usam os sufixos configuraveis ao resolver os nomes dos campos.
- O comportamento de pontos (`x` e `y`) foi mantido sem alteracoes.

2.2.5.5

- Corrigida chamada de leitura dos sufixos no `SettingsPlugin` para compatibilidade com `InputFieldsWidget.get_value`, evitando erro em runtime ao salvar as preferencias.

2.2.5.6

- Adicionada nova ferramenta de dialogo `VectorToSvgPlugin` na categoria `FOLDER`, registrada no `ToolRegistry` com `cadmus_icon` e exibicao em toolbar e menus.
- Implementada a UI inicial do `VectorToSvgPlugin` herdando de `BasePluginMTL`, composta apenas no `_build_ui()` com `WidgetFactory.create_layer_input`, tres `create_color_button` e um `create_checkbox_grid`.
- Registrado novo `ToolKey.VECTOR_TO_SVG` para logging e identificacao da ferramenta.

2.2.5.7

- Corrigido import de `QgsMapLayerProxyModel` no `VectorToSvgPlugin`, trocando `qgis.gui` por `qgis.core` para compatibilidade com QGIS 3.16.

2.2.5.8

- Refinado o layout do `ColorButtonWidget` para melhor alinhamento horizontal entre titulo, seletor de cor, input hexadecimal e botao de copiar.
- Removida a largura fixa pequena do titulo na `WidgetFactory.create_color_button`, reduzida a altura visual do input hexadecimal e ajustado o botao de copiar para respeitar melhor os tamanhos configurados.
- Simplificado o estilo de `Styles.color_button_widget()` para evitar que o QSS force alturas e tamanhos minimos conflitantes com a configuracao aplicada pela factory.

2.2.5.9

- Padronizados novos mĂ©todos de `QgisMessageUtil`: `ask_overwrite`, `show_styled_message_bar`, `show_progress_message_bar`, `update_progress`, `clear_message_bar`, `show_level_message_bar` e `show_message_bar_with_button` foram convertidos em mĂ©todos estĂˇticos.
2.2.5.10

- Implementada a execucao completa do `VectorToSvgPlugin`, com exportacao de SVG unico ou um SVG por feicao, respeitando `somente feicoes selecionadas`, fundo transparente, exibicao opcional de borda e rotulo, e abertura do resultado na pasta de saida.
- A exportacao agora usa `ProjectUtils` para resolver pasta do projeto e validar a camada, `VectorLayerGeometry.get_selected_features()` para materializar apenas a selecao quando o checkbox estiver ativo e mantem o renderer da camada original para desenhar as geometrias no SVG.
- Adicionada a regra de nomeacao para multiplos SVGs: usa o campo `Name` quando existir e tiver valor; caso contrario, usa o nome da camada do QGIS com sufixo incremental `_1`, `_2`, `_3`...
- Incluida persistencia da opcao `Gerar SVG para cada feicao` e ajuste do plugin para salvar os arquivos na pasta escolhida com nomes sanitizados e sem sobrescrever arquivos existentes.
