2.2.1.6

- Reorganizada interface do SettingsPlugin: adicionados dois widgets colapsáveis - "Geral" (botão de preferências, idioma, precisão de campos vetoriais, limiar assíncrono) e "Plugin de Calculos Vetoriais" (método de cálculo vetorial) para melhor organização da UI.

2.2.5.1

- Corrigido bug de inicialização de log em `LogUtils`: anteriormente usava `Path.cwd()` (ex: `C:\Windows\system32`) quando não inicializado, causando `PermissionError` ao criar pasta `log`.
- Implementado fallback seguro para `plugin_root` via `Path(__file__).resolve().parents[3]` e, se necessário, `Path.home()/Cadmus/log`.
- Ajustado `LogUtils._write_event` para continuar sem arquivo de log e enviar mensagem por `QgsMessageLog`/stderr em casos de falha de criação de arquivo.

2.2.5.2

- Adicionados no `SettingsPlugin`, dentro de `calc_collapsable`, dois campos configuráveis inicialmente para nomes de área cartesiana e elipsoidal com `WidgetFactory.create_input_fields_widget`.
- As preferências desses campos passaram a ser carregadas e salvas em `ToolKey.VECTOR_FIELDS`.
- Incluída validação para impedir salvamento quando os dois valores configurados forem iguais, exibindo alerta com `QgisMessageUtil`.

2.2.5.3

- Ajustados os dois campos do `SettingsPlugin` para representar sufixos, não nomes completos de campo: `Sulfixo Cartesiano` com padrão `""` e `Sulfixo Elipsoidal` com padrão `"_eli"`.
- As preferências em `ToolKey.VECTOR_FIELDS` passaram a usar as chaves `cartesian_suffix` e `ellipsoidal_suffix`.
- Mantida a validação para impedir salvamento quando os dois sufixos forem iguais, agora com mensagem alinhada ao novo comportamento.
