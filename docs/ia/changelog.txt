2.3.6.1.4 2026/04/06
[2.3.6.1.4] CreateProjectPlugin: When the current project is unsaved, it is now saved with its current layers into the new project folder; when already saved, the new blank project is created and opened in a separate QGIS window, preserving the current window unchanged.
2.3.6.1.5 2026/04/06
[2.3.6.1.5] ToolRegistry/SettingsPlugin: Replaced local hard-coded UI strings with TranslationManager STR keys and added the required entries only in Strings_pt_BR.

[2.3.6.1.6] SettingsPlugin/CreateProjectPlugin: Removed the duplicate default CRS label string by reusing DEFAULT_CRS directly and migrated CreateProjectPlugin user-facing texts to TranslationManager keys added only in Strings_pt_BR.
2.3.6.1.7 2026/04/07
[2.3.6.1.7] CreateProjectPlugin: Added automatic Google Satellite XYZ basemap insertion in newly created projects (both unsaved current project flow and new-window flow), with safe fallback when tile layer creation is invalid.
2.3.6.1.8 2026/04/07
[2.3.6.1.8] CreateProjectPlugin: Fixed Google Satellite XYZ source URI by percent-encoding nested URL parameters (`x/y/z`) to prevent tile source parsing issues that caused empty basemap layers.
2.3.6.1.9 2026/04/07
[2.3.6.1.9] CreateProjectPlugin: Applied default project CRS from SYSTEM preferences (`default_crs_authid`) during project creation, with validation and fallback to `EPSG:4326` when missing or invalid.

[2.3.6.1.10] CreateProjectPlugin/ProjectUtils: Switched basemap to Google Hybrid (XYZ), added conditional auto-load of `line.gpkg` only for scenarios 2 and 3, and introduced file-extent canvas centering utility in ProjectUtils for automatic focus on the reference line.
2.3.6.1.11 2026/04/07
[2.3.6.1.11] CreateProjectPlugin/ExplorerUtils/VectorLayerSource: line.gpkg is now copied into each new project's `vectors` folder before loading, and the loaded reference layer now uses the copied file path via VectorLayerSource; extent centering also targets the copied file.
2.3.6.1.12 2026/04/07
[2.3.6.1.12] CreateProjectPlugin/core.ui.dialogs: Extracted default-folder and project-name dialogs into dedicated dialog classes under `core/ui/dialogs`, and refactored `execute()` into focused steps (resolve folder, resolve name, create structure) to reduce mixed responsibilities without changing creation behavior.
2.3.6.1.13 2026/04/07
