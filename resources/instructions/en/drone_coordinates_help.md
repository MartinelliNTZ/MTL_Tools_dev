# 📘 Get Drone Coordinates (MRK) – User Manual  
Tool from the **Cadmus** package for generating **drone flight points and trajectories** from **MRK** files (DJI).

---

## 📌 What does this tool do?

This tool reads **MRK files generated by DJI drones** and allows you to:

- Generate **photo points** from the flight
- Generate the **drone trajectory (line)**
- Optionally **cross‑reference points with JPG photo metadata**
- Save results to a file or create **temporary layers**
- Apply **QML styles** automatically
- Process large volumes without freezing QGIS

---

## 🧩 Main components

### ✔ MRK folder
Defines the folder where the `.mrk` files are located.

- Can contain **subfolders**
- There can be **multiple MRKs**

---

### ✔ Search subfolders
When checked, the tool searches for `.mrk` files **recursively in all subfolders** of the selected folder.

⚠ **Caution:**  
- If unchecked, only MRKs in the **root folder** will be processed
- This option is ignored if individual files are selected directly

---

### ✔ Merge all MRKs
When activated, all found MRK files are **aggregated into a single flight**, generating:

- **A single set of points** (with all waypoints combined)
- **A single trajectory** (with the complete unified flight path)

If unchecked, each MRK is processed **independently**, creating multiple point and trajectory layers.

---

### ✔ Cross‑reference with photo metadata
Allows you to enrich MRK points with information extracted from the **flight JPG photos**, such as:

- **Photo file name**
- **File size** (bytes)
- **File modification date**
- **Image dimensions** (width × height)
- **EXIF metadata**: ISO, aperture (f‑stop), focal length, camera make, etc.

Requires that the photos be available **in the same folder** as the MRK or in subfolders.

⚠ **Caution:**  
- Not recommended for **large photo volumes** (>5000 images), because processing is asynchronous and may take minutes
- If a photo is not found, the MRK point will be created **without photo metadata** (processing continues)

---

## 📍 MRK Points

### ✔ Save points to file
Allows you to save the points as:

- Shapefile (`.shp`)
- GeoPackage (`.gpkg`)
- GeoJSON (`.geojson`)
- KML (`.kml`)

If unchecked, the points will be created as a **temporary layer**.

---

### ✔ Apply style (QML) to points
Allows you to select a `.qml` file to automatically style the point layer.

---

## 🧭 Drone Trajectory (Line)

### ✔ Save trajectory to file
Allows you to save the drone trajectory as:

- Shapefile
- GeoPackage
- GeoJSON
- KML
- CSV

If unchecked, the trajectory will be created as a **temporary layer**.

---

### ✔ Apply style (QML) to trajectory
Allows you to apply a `.qml` style to the drone track.

If no QML is provided, the tool may apply a **default plugin style**.

---

## ▶ How to use

### 1. Open the tool  
Menu → **Cadmus** → **Precision Agriculture** → *Get Drone Coordinates*

---

### 2. Select the MRK folder
Click **…** and choose the folder where the `.mrk` files are located.

---

### 3. Set desired options
Check as needed:

- Search subfolders
- Merge MRKs
- Cross‑reference with photos
- Save points
- Save trajectory
- Apply styles

---

### 4. Define output paths (optional)
If you choose to save to a file:

- Select the location and format
- Choose QML files if desired

---

### 5. Click **Run**
The tool will:

1. Read all found MRKs  
2. Create the points  
3. Create the trajectory  
4. Cross‑reference with photos (if enabled)  
5. Add the layers to the project  

Progress and messages are automatically logged.

---

## 🔄 Persistence of Settings

All chosen options are automatically saved in **QGIS preferences** for the next session:
- Last selected folder / files
- State of checkboxes (search subfolders, merge MRKs, etc.)
- Output paths and QML styles

⚠ **Note:**  
Only the **first path** (folder or file) is remembered. If multiple files were selected, only the first one will be restored.

---

## ℹ️ Important Notes

- **Asynchronous processing:** Especially when cross‑referencing with photo metadata, the tool runs in a **background thread** to avoid freezing QGIS
- **Automatic reprojection:** KML files are automatically reprojected to **EPSG:4326** (WGS84)
- **Temporary layers:** If no output path is provided, points and trajectories are created as **memory layers** (temporary)
- **MRK not found:** If the folder contains invalid or corrupted `.mrk` files, the tool will report warnings but continue processing valid ones
- **Photos not found:** If you enable "Cross‑reference with photos" but some JPGs are missing, the tool will create the MRK point without photo metadata (it will not fail)
