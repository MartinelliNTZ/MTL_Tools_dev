ARQUITETURA DE WIDGETS PARA USAR COM WIDGETFACTORY

Objetivo
Remover completamente o uso direto de qgis.PyQt.QtWidgets dentro dos dialogs.
Os dialogs apenas montam a interface utilizando a API da WidgetFactory e widgets reutilizáveis.

Regra arquitetural

1. Dialogs

* NÃO podem importar QtWidgets
* NÃO criam layouts
* NÃO criam botões, labels, lineedits etc
* Apenas usam a API da factory

2. WidgetFactory

* Pode usar QtWidgets
* Responsável por criar widgets simples
* Pode combinar widgets e aplicar estilos
* Pode adicionar separadores, espaçamento e layout padrão

3. Widgets exclusivos

* Usados quando o componente tem lógica própria
* Podem encapsular vários widgets internos
* São reutilizáveis entre ferramentas

==================================================
WIDGETS QUE DEVEM EXISTIR NA FACTORY
====================================

1. ReadOnlyField

Uso:
mostrar valores calculados que podem ser copiados.

Estrutura visual:

Label | Campo readonly | Botão copiar

Casos de uso:

Latitude decimal
Longitude decimal
Latitude DMS
Longitude DMS
UTM X
UTM Y
Altitude
Qualquer resultado numérico

Responsabilidades:

* label do campo
* lineedit readonly
* botão copiar opcional
* método set_value(valor)

Exemplo visual:

Latitude (Decimal) | -10.23423423 | [Copiar]

Benefício:

Elimina no dialog:

_line_edit()
_line()
botões copiar individuais

==================================================
2. InfoField
============

Uso:
mostrar informação simples apenas com label.

Estrutura:

Label: Valor

Casos de uso:

Zona UTM
EPSG
Município
Estado
Região
País

Exemplo visual:

Zona UTM: 23S
Estado: Tocantins

Responsabilidades:

* label fixo
* valor dinâmico
* método set_value(valor)

==================================================
3. GroupSection
===============

Uso:
container com título para agrupar widgets.

Substitui QGroupBox dentro do dialog.

Estrutura:

[TÍTULO]

conteúdo...

Exemplos no dialog:

WGS 84 (EPSG:4326)

UTM SIRGAS 2000

Altimetria

Localização Administrativa

Responsabilidades:

* título
* layout interno
* padronizar margens
* padronizar spacing

==================================================
4. CopyBlockButton
==================

Uso:
copiar várias informações de uma vez.

Exemplos no dialog:

Copiar WGS completo
Copiar UTM completo
Copiar localização completa

Responsabilidades:

* botão
* copiar texto para clipboard
* callback custom

==================================================
WIDGETS OPCIONAIS MAS MUITO ÚTEIS
=================================

Esses widgets encapsulam blocos maiores.

Devem ser criados apenas se forem reutilizados em mais ferramentas.

==================================================
5. CoordinateDisplayWidget
==========================

Mostra coordenadas WGS.

Conteúdo interno:

Latitude decimal
Longitude decimal
Latitude DMS
Longitude DMS
Zona UTM

Uso comum em:

MapTools
Identify tools
Plugins de geocodificação

==================================================
6. UTMDisplayWidget
===================

Mostra coordenadas UTM.

Conteúdo:

EPSG
Easting (X)
Northing (Y)
Zona UTM

==================================================
7. AdministrativeLocationWidget
===============================

Mostra informações administrativas.

Conteúdo:

Município
Região intermediária
Estado
Região
País

Esse widget costuma ser reutilizado em:

Reverse geocoding
Identify
Map click tools

==================================================
8. AltitudeResultWidget
=======================

Mostra altitude.

Na maioria dos casos não precisa widget exclusivo.
Pode usar ReadOnlyField.

==================================================
MÉTODOS QUE A FACTORY DEVE TER
==============================

Para permitir dialogs sem QtWidgets.

Campos:

create_readonly_field

create_text_field

create_number_field

create_combo_field

Exibição:

create_info_field

create_status_label

create_warning_label

Containers:

create_group_section

create_vertical_block

create_horizontal_block

Utilidades:

create_copy_button

create_separator

create_spacer

==================================================
COMO O DIALOG DEVE FUNCIONAR
============================

O dialog não constrói UI manualmente.

Ele apenas solicita blocos da factory.

Fluxo conceitual:

1. criar layout principal via factory

2. criar grupos:

WGS
UTM
ALTITUDE
LOCALIZAÇÃO

3. dentro de cada grupo adicionar campos da factory

Exemplo lógico:

grupo WGS

latitude decimal
longitude decimal
latitude DMS
longitude DMS
zona UTM

grupo UTM

EPSG
X
Y

grupo altitude

Altitude

grupo localização

Município
Região intermediária
Estado
Região
País

==================================================
BENEFÍCIOS DA ARQUITETURA
=========================

1. Dialogs ficam muito simples
   apenas orquestram widgets

2. Nenhum QtWidget dentro dos dialogs

3. UI consistente em todo plugin

4. Widgets reutilizáveis

5. Mudança de estilo centralizada na factory

6. Facilita manutenção em plugins grandes

==================================================
RESUMO PRÁTICO
==============

Widgets essenciais:

ReadOnlyField
InfoField
GroupSection
CopyBlockButton

Widgets opcionais (para reutilização):

CoordinateDisplayWidget
UTMDisplayWidget

Responsabilidade final:

Dialog → apenas usa factory
Factory → cria e organiza widgets
